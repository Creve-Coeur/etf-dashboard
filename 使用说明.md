# ETF 持仓看板脚本使用说明

现在只保留一个主脚本：

```text
获取每日持仓情况_ver3.0.py
```

脚本已按 Spyder 的 `#%%` 分成多个段落。直接运行整份文件会自动执行完整流程：打开券商网站、等待下载、刷新网页数据、部署云端。

如果想分步骤排查，可以只运行某个 `#%%` 段。脚本最下面“分步骤执行区”里的每个 cell 都是直接执行语句，不需要取消注释，也不需要设置开关。运行到哪个 cell，就执行哪个功能。手动运行得到的 `benchmark_map`、`found_file`、`excel_path`、`dashboard_data`、`deploy_result`、`workflow_result` 等变量都会留在变量浏览器里。

## 常用分步骤运行

在脚本最下面的“分步骤执行区”直接运行对应 cell。

检查全部指数：

```python
benchmark_map, benchmark_errors = check_benchmark_data()
```

检查单个指数：

```python
benchmark_map, benchmark_errors = check_benchmark_data(index_name="沪深300")
```

只刷新网页数据：

```python
dashboard_data = refresh_data_json_from_excel(excel_path, benchmark_map=benchmark_map, benchmark_errors=benchmark_errors)
```

只打开券商网站：

```python
start_time = open_broker_website()
```

等待并捕获新下载文件：

```python
WAIT_MAX_SECONDS = 20
found_file = wait_for_new_export(start_time, max_wait_time=WAIT_MAX_SECONDS)
```

移动下载文件：

```python
excel_path = move_export_to_project(found_file)
```

只部署云端：

```python
deploy_result = deploy_to_cloud()
```

完整流程手动运行，等同于直接运行整份脚本：

```python
workflow_result = {
    "benchmark_map": benchmark_map,
    "benchmark_errors": benchmark_errors,
    "start_time": start_time,
    "found_file": found_file,
    "excel_path": excel_path,
    "dashboard_data": dashboard_data,
    "deploy_result": deploy_result,
}
```
